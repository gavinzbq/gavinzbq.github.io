HOWTO: Package Software & Submit to PyPI

synonym-0.1.2 has just been released. Two features are added:

  • enable / disable colorized output.
  • use cache by default for faster access while enable clear cache functionality.

In this article I would like to conclude my experience with packaging Python project by making a curated list.

What is packaging all about?

Have you ever wondered a library or application can be installed by one single pip command? In short, we can get our favorite applications by pip install synonym is because synonym is indexed at Python Packaging Index (PyPI).

The purpose of packaging is to distribute your work (code to be specific). You'll either want to create a library to be imported or a standalone application to be used by other developers. And you don't want people install your work by configure their development environment and install dependencies step by step. It's not very nice.

Packaging is to generate a bundle which set up the environment automatically and correctly. That is what packaging is all about.

For details, please do read Packaging and Distributing Projects to understand the basics of how to configure and package Python projects.

I have never packaged a project.

All right, I want to share my code. And what to do next?

Don't DIY. Unless you enjoy manually typing setup.py. I use cookiecutter and you may consider it as well. It generates project templates which can be customized later. For Python projects, use the cookiecutter template called pypackage. Follow the tutorial and you will be square. Do create accounts at testpypi and pypi beforehand.

Release on PyPI to make your code pipable.

How to submit a package to PyPI is a great post albeit it's a little bit outdated. As a consequence of PyPI migrating to PyPI.org, the Creat .pypirc configuration file session needs to be altered. Refer to this instruction.

Releases after the first one?

This article provided the workflow and checklist before releasing a new version. I would recommend doing it once, then consider a automation tool such as Bumpversion. Use DRY, avoid DIY when things get tedious.

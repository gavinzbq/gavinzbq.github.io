How I created this website

The website is generated by Pelican. It is a static site generator written in Python. I will describe the building process briefly, and will make a list of useful materials I refered to. Since I have been playing with it for merely three weeks, you will not see a comprehensive review here. And you will not see a step-by-step tutorial either because there have already been many good ones. I recommend Making a Static Blog with Pelican.

Prerequisite

  • Python. A little bit experience is required, but do not start learning the language from here. The more fluent you are, more hackable the framework would be. I myself started learning Python from: * Google's Python Class.
  • HTML, CSS and JavaScript It is useful to learn about Internet, World Wide Web. HTML is the webpages' conttent, and CSS is in charge of styling. Good learning lessons can be found here: MarkSheet. My favorite MarkSheet's feature is each tutorial lasts no longer than ten minutes.
  • A healthy mindset Open source softwares are not easy to use. The beauty of open source is not only that they are free, but we will find we are not the only person using it. And this never ending learning and communicating process is very rewarding. Programming can be tedious, we should use it as a tool to avoid more tedious everyday tasks for later on.

Things I want to stress and hurdles I encountered

Most parts of frustration comes from manipulating the theme. I have no better advice other than do spend some time conducting experiment with HTML template files and CSS files.

If you would like to host articles with different topics, just assign "category" metadata to them. For example, articles under "Blogs" navigation bar have category of "Blog", and those under "Gallery" belong to "Gallery" category.

I found photos very handy to process images. "Photos" is a powerful plugin if you want to post bunches of photos like I did, it can resize images / generate thumbnails / embed images and even whole gallery in articles. The documentation is very helpful.

"photos" plugin would be perfect if it could generate thumbnails according to the exact size I specified. For the matter of style and taste, square thumbnails are more desirable yet "photos" can only provide those whose aspect ratio inherits from their original images. Therefore I turned to another plugin specialize in processing thumbnails named thumbnailer. I would like to add this feature to "photos" if I have time.

In general, I found Pelican, its associated themes and plugins are fairly customizable. Bear in mind that diving in the official documentation never hurts, especially when Google answers are exhausted.

Enjoy hacking.
